# FB-20260528-002: Contacts for SECURITY.md and CODE_OF_CONDUCT.md are undecided

- Target design commit: `b129d0ae1e7d91de5cdd406b8cd463f746efe8b3`

## Documentation Blocker

The latest design requires `SECURITY.md` to include a real private contact for
vulnerability reports. It also requires `CODE_OF_CONDUCT.md` to replace the
contact field with a contact that can actually handle enforcement reports.

The design and public repository documentation do not confirm a private
security report contact or a Code of Conduct enforcement contact that can be
published. A personal email address from a git signature cannot be treated as a
maintainer-approved public contact.

## Written and Omitted Documentation

The following OSS publication documents were created:

- `README.md`
- `LICENSE`
- `CONTRIBUTING.md`
- `CHANGELOG.md`
- `.github/ISSUE_TEMPLATE/bug_report.yml`
- `.github/ISSUE_TEMPLATE/feature_request.yml`
- `.github/pull_request_template.md`

`SECURITY.md` and `CODE_OF_CONDUCT.md` were not created because doing so would
require choosing real contacts without maintainer approval. `CONTRIBUTING.md`
only states that security issues must not be reported in public issues and
that `SECURITY.md` needs a private contact before OSS publication.

## Requested Design Clarification

Please add the contacts that are approved for publication to the design:

- the private contact for security reports;
- the Code of Conduct enforcement contact;
- whether the first-response target and responsible owner can use the same
  contact or need separate contacts.

After this information is decided, `SECURITY.md` and `CODE_OF_CONDUCT.md` can
be added.
